# VASSAL Uber Jar Build

## Overview

The VASSAL project now supports building an "uber jar" (also known as a "fat jar") that contains all dependencies bundled into a single executable JAR file. This makes it easier to distribute and run the application without requiring separate dependency management.

## Building the Uber Jar

### Prerequisites
- Java 11 or higher
- Maven 3.5 or higher

### Build Command

To build the uber jar, run:

```bash
mvn clean package
```

This will create both the regular JAR and the uber JAR in the `vassal-app/target/` directory:

- `vassal-app-<version>.jar` - Regular JAR (~5.5 MB) - requires dependencies in classpath
- `vassal-app-<version>-uber.jar` - Uber JAR (~19 MB) - contains all dependencies

### Build Output

```
vassal-app/target/
├── vassal-app-3.8.0-SNAPSHOT.jar         # Regular JAR
├── vassal-app-3.8.0-SNAPSHOT-uber.jar    # Uber JAR with all dependencies
├── vassal-app-3.8.0-SNAPSHOT-sources.jar # Source code
└── vassal-app-3.8.0-SNAPSHOT-javadoc.jar # JavaDoc
```

## Running the Uber Jar

The uber jar is self-contained and can be executed directly:

```bash
java -jar vassal-app/target/vassal-app-3.8.0-SNAPSHOT-uber.jar
```

### Advantages of the Uber Jar

1. **Single File Distribution**: All dependencies are bundled, making distribution simpler
2. **No Dependency Management**: Users don't need to manage classpaths or external libraries
3. **Easy Deployment**: Can be deployed as a single artifact
4. **Consistent Environment**: Guarantees all dependencies are at the correct versions

### What's Included

The uber jar includes all runtime dependencies:
- Logback (logging framework)
- Apache Commons libraries (IO, Lang3, Codec)
- Apache Batik (SVG processing)
- MiGLayout (Swing layout)
- HTTP Client libraries
- All other project dependencies

Total: ~11,900 files from all dependencies

## Technical Details

### Maven Shade Plugin Configuration

The uber jar is created using the Maven Shade Plugin with the following features:

- **Main Class**: `VASSAL.launch.ModuleManager`
- **Resource Transformers**:
  - Manifest transformer (sets main class)
  - Services transformer (merges META-INF/services files)
  - Apache license transformer
- **Filters**: Excludes signature files (*.SF, *.DSA, *.RSA) to prevent security exceptions

### Configuration Location

The Maven Shade Plugin configuration is located in:
- `vassal-app/pom.xml` - Plugin execution configuration
- `pom.xml` - Plugin version management in pluginManagement section

## Troubleshooting

### "Invalid signature file" Error

If you encounter signature verification errors, ensure the plugin filters are configured to exclude signature files. This is already configured in the current setup.

### "No main manifest attribute" Error

The uber jar should have the main class automatically set. If this error occurs, verify that the ManifestResourceTransformer is properly configured.

### Large File Size

The uber jar is significantly larger (~19 MB) than the regular jar (~5.5 MB) because it contains all dependencies. This is expected and normal for uber jars.
